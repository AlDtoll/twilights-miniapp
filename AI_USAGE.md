# Twilights Mini App — инструкция для ИИ-бота

## Что это

Telegram WebApp для мини-игр: выбор из вариантов и блэкджек (21).  
URL: `https://aldtoll.github.io/twilights-miniapp/`  
Репо: `AlDtoll/twilights-miniapp`

## Как запустить игру

Скрипт находится в репо `twilights-world`:

```bash
cd /home/claudeuser/twilights-world
export TELEGRAM_BOT_TOKEN=<токен_бота>

# Выбор из вариантов
python3 twilights/minigames/send_miniapp.py <chat_id> "<текст сообщения>" choice \
  --prompt "<вопрос игроку>" \
  --options "<вариант1>|<вариант2>|<вариант3>"

# Игра в 21 (блэкджек)
python3 twilights/minigames/send_miniapp.py <chat_id> "<текст сообщения>" blackjack
```

## Как получить результат

После того как игрок сделал выбор и нажал «Подтвердить», мини-апп отправляет боту **обычное текстовое сообщение** вида:

```
🎮 Игрок сделал выбор:
→ <текст выбранного варианта>
{"game":"choice","choice":<индекс>,"text":"<вариант>"}
```

Или для блэкджека:
```
🎮 Игрок сделал выбор:
→ 21: счёт 19 vs 17 — победа
{"game":"blackjack","playerScore":19,"dealerScore":17,"outcome":"win"}
```

`outcome`: `win` / `lose` / `draw`

Это **обычное сообщение** — бот получает его через стандартный polling без каких-либо специальных обработчиков.

## Как реагировать на результат

После получения JSON с результатом — применить его в контексте сцены:

- `choice`: игрок выбрал вариант → развернуть нарратив по выбранному пути
- `blackjack`: исход состязания → применить механику победы/поражения/ничьей из `contest_core.md`

## Технические детали

- Кнопка отправляется как **ReplyKeyboard** (обычная клавиатура внизу), не inline
- Токен бота и `chat_id` передаются в зашифрованном виде в URL параметре `?data=base64`
- Мини-апп делает прямой fetch на `api.telegram.org` — никаких серверов на VPS не нужно
- После подтверждения мини-апп закрывается автоматически
